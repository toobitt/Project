代码仓库使用说明：

目录说明：
trunk    -- 主开发目录
tags     -- 版本发布目录，此目录下不允许提交任何修改
branches -- 开发分支目录

流程说明：
1. 新功能可能会影响到主版本的，统一在branches下的feature目录创建分支开发，开发测试没问题之后再合并到主版本（trunk）中
2. 对tags中版本的修复需要先在branches下的bugfix目录里创建新的分支开发，bug修复之后再release新的小版本到tags目录中

注意事项：
1. 提交代码时请负责任的写下提交说明
2. 如果在redmine中有关联Issue，请使用refs #XXX: (XXX=Issue号)关键字，这样本次提交可以关联到redmine
3. 提交前请认真review自己的代码
4. 任何时候都别忘记先update再提交
